1. Для сохранения данных делаем проброс папок через volumes
    - Хранилище для docker-контейнеров в папке storage/docker/<name>
    - Для конфигурационных файлов в корне проекта папка docker

2. Чтобы не привязываться к папке проекта в docker контейнерах, в файле .env зададим имя COMPOSE_PROJECT_NAME
3. Для быстрого запуска команд создать файл в прокте Makefile и прописать соответствующие конструкции
